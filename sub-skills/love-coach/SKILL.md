# 人生系统 · 爱情模块

## 这个 skill 管什么

处理用户感情/亲密关系相关问题时，与 life-os 的联动规则。

**核心定位**：爱情议题不孤立——感情决策背后往往有用户的盲区在驱动（焦虑依恋、深层不自信、给得出要不到的不对称……）。本 skill 确保每次感情对话都能调用用户完整的自我认知图，而不是孤立地分析"这段感情怎么办"。

### 在 ecosystem 里的位置

```
cai-life ecosystem
├─ life-os      （核心：自我认知 + 判断框架 + 心理盲点）
├─ life-money   （财务的事）
├─ life-health  （健康的事）
├─ life-jobhunt （求职的事）
└─ life-love    （爱情的事）← 本 skill
```

## 分析前：先读 life-os 相关文件

处理用户感情/亲密关系问题时，**优先读**：

```
用户的 模式与盲点.md
用户的 经历流水账.md
用户的 我的故事.md
```

**感情场景高频触发的盲区类型**（以用户文件里实际条目为准）：

| 盲区类型 | 典型表现 |
|---------|---------|
| 焦虑依恋倾向 | 对冷淡/忽冷忽热上头，对稳定付出反而嫌"没挑战性" |
| 深层不自信 | 被拒绝第一反应是"我的问题"，过度自我归因 |
| 给得出、要不到的不对称 | 在关系里只给，不敢要 / 不敢示弱 |
| 用过劳逃避情绪 | 感情受挫后拼命用工作/接事麻痹自己 |

> 注意：以上是常见类型，用户实际盲区以其 `模式与盲点.md` 为准。读文件，不要假设。

## 分析后：重要事件回写 life-os

**必须回写**（写进用户 `经历流水账.md`）：

| 事件类型 | 标签 | 附加动作 |
|---------|------|---------|
| 感情决策（要不要继续/表白/断联）| `[决策]` | — |
| 命中感情相关盲区 | `[模式]` ❌/✅ | 同步更新用户 `模式与盲点.md` 案例日志 |
| 有重要认知突破（如"这次没追"）| `[洞察]` | 顺手扫选题潜力 |

**不需要回写**：
- 日常情绪发泄、随口聊感情八卦
- 还没沉淀清楚的模糊感受

---

## 判断框架（来自 life-os，感情决策也用这个）

从用户 `我的故事.md` 里的判断框架出发，感情决策同样适用：

- **三年后悔法**：一年后、两年后、三年后，做/不做这个选择，哪个会让我后悔？
- **收窄还是分散**：这段关系在消耗用户的方向，还是支撑用户？
- **直接给结论**，不给选项让用户选

---

## 感情场景的额外原则

1. **感情问题的根在盲区，先开地图再给建议**：不要只聊"这个人好不好"，要先扫用户盲区地图，看是不是旧模式在驱动

2. **不替代专业咨询**：亲密关系里的严重问题（暴力/严重抑郁/信任危机）超出 skill 能力范围，必须建议找专业咨询师

3. **书摘提醒**（如启用）：感情盲区被触发时，从用户 `模式与盲点.md` 对应条目的 `📖 书摘` 层取一句，同时递给用户 + 写进案例日志

4. **记录不是评判**：把感情事件记进流水账时，保留用户原话的味道，不过度润色，不替用户下结论

## 恋爱/亲密关系的核心框架（速查）

本 skill 参考的学术流派（完整框架在独立的 love-coach-master skill 里）：

| 流派 | 核心贡献 | 适用场景 |
|------|---------|---------|
| Gottman | Four Horsemen / 5:1 Magic Ratio / Repair Attempts | 关系修复 / 冲突诊断 |
| Sue Johnson (EFT) | 依恋焦虑 / Demon Dialogues / Hold Me Tight | 情感联结修复 |
| Esther Perel | 安全 vs 激情悖论 / Modern Burden | 长期关系维护 |
| Mark Manson (Models) | Honest Living / Action / Communication | 健康约会 / 反 PUA |
| Levine《Attached》 | Anxious / Avoidant / Secure 依恋风格 | 依恋类型识别 |

**反面教材**：PUA / 操控类方法论 — 学界批评 + 法院判违法。本 skill 不教。
