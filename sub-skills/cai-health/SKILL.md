# 人生系统 · 健康模块

## 这个 skill 管什么

用户的健康相关全流程：
- **体检后**：把指标结构化、跟历次比、标异常项
- **平时**：症状日志（看医生时带过去）
- **看完医生**：就诊归档
- **横向**：积累健康知识库
- **决策**：要不要做 X 检查 / 治疗 / 该不该停下来——联动 life-os

### 和 life-os 的边界

| skill | 管什么 |
|-------|------|
| life-health | 健康的「事」——指标、就诊、症状、知识 |
| life-os | 健康背后的「我」——「用过劳逃避情绪」「为赚钱牺牲健康」「健康/数字回避」等心理盲点 |

### 在 ecosystem 里的位置

```
cai-life ecosystem
├─ life-os      （核心：自我认知 + 判断框架 + 心理盲点）
├─ life-money   （财务的事）
├─ life-health  （健康的事）← 本 skill
├─ life-jobhunt （求职的事）
└─ life-love    （爱情的事）
```

每个子 skill 管自己领域的"事"，life-os 管所有领域背后的"我"。

### 回写 life-os 流水账的触发标准

**必须回写**（写进用户 `经历流水账.md`，标 `[决策]` 或 `[模式]`）：
- 体检发现异常指标 + 有后续决策（做/不做检查/治疗）
- 就诊后医生给了重要建议或诊断
- 识别到「用过劳逃避情绪」「为赚钱牺牲身体」等盲区再现 ❌ 或突破 ✅
- 第一次主动停下来休息（突破「Yes惯性」的身体版）

**不需要回写**（留在 life-health 自己的文件里）：
- 日常症状记录（头疼、肩颈酸等）
- 普通保健品选购
- 体检数据存档（无异常项）

## 场景一：整理体检报告

**触发**：「我体检完了」「整理体检报告」「最近做了体检」+ 用户上传 PDF / 贴关键指标

**执行**：
1. 接收体检报告（PDF / 文本 / 关键指标列表）
2. 提取关键指标，按系统分类：血液 / 生化 / 影像 / 心电 / 妇科 等
3. 读上一次 `体检记录.md` 的数据，做趋势对比（↑/↓/→）
4. 标异常项 ⚠️ —— 偏离参考范围的指标
5. 把新一次体检追加到 `体检记录.md` 顶部（倒序）
6. **异常项 → 单独写进"待跟进"区**，关联待复查
7. 原始 PDF 复制到 `体检原件/YYYYMMDD-医院名.pdf`
8. **检查心理状态**：用户看体检报告时有抗拒/逃避吗？有的话触发联动 life-os 的「健康/数字回避」（财务回避的姊妹盲点）

**关键原则**：
- 数字要准确，不要四舍五入
- 趋势用 ↑/↓/→ 直接显示
- 异常项必须 ⚠️ 高亮
- **不解读病情**（AI 不是医生），只整理事实

## 场景二：症状记录

**触发**：「我哪里不舒服」「记一下症状」「最近 XX 部位有点 XX」

**执行**：
1. 读用户 `症状日志.md`
2. 把症状整理成 1-2 句话
3. 加在日志最上面（倒序），格式：
   ```
   YYYY-MM-DD · 部位/类型 · 感受 · 持续多久 · 触发因素
   ```
4. 如果用户提到要看医生 → 提示"看医生前我帮你把症状整理一份带过去"

**关键原则**：
- 不诊断（AI 不是医生）
- 追问关键细节：持续多久？触发因素？严重程度？
- **严重症状（剧痛、出血、呼吸困难等）→ 直接说"建议立刻就医"**，不要在 skill 里磨

## 场景三：就诊归档

**触发**：「我去看医生了」「整理就诊记录」「医生说...」

**执行**：
1. 询问：哪家医院 / 哪个科 / 医生是谁 / 主诉是什么 / 诊断 / 医嘱 / 用药 / 复查时间
2. 整理成结构化条目，加在用户 `就诊记录.md` 最上面
3. 用药 → 同步到 `健康档案.md` 的"长期用药"（如果是长期）
4. 复查时间 → 单独列出来，提醒用户加到看板 / 日历
5. 关联到 `症状日志.md`：本次主诉对应的症状条目末尾标 ✅ 已就诊

## 场景四：健康知识库

**触发**：「记一下健康知识」「医生教我 XX」「朋友说 XX」「读到一个健康知识」

**执行**：
1. 读用户 `健康知识库.md`
2. 整理成知识卡片（同 life-money 财务知识库格式）
3. 卡片要素：**是什么 / 影响 / 怎么做 / 注意 / 来源+日期**
4. 没完全搞懂的 → 丢「待搞清楚」区

## 场景五：健康决策（联动 life-os ⭐）

**触发**：「要不要做 XX 检查」「这个治疗值得吗」「该不该停下来休息」「这个保健品要不要买」「这个手术要不要做」

**执行**：
1. **读 life-health 资料**：用户 `体检记录.md` + `就诊记录.md` + 相关 `健康知识库.md` 卡片
2. **读 life-os 资料**（关键）：
   - 用户 `模式与盲点.md` —— 重点扫这些常见健康相关盲区：
     - 「用过劳逃避情绪」（已识别盲区，如有）
     - 「精力分散倾向」「Yes 惯性 / 高估精力储备」
     - 「健康/数字回避」
   - 用户 `我的故事.md` —— 拿判断框架 + 用户健康相关的核心叙事
3. **直接给结论**，不给选项
4. 用判断框架：
   - **三年后悔法**：一年/两年/三年后回看，做和不做哪个会让我后悔？
   - **离健康的自己更近还是更远**？
   - **是不是「用过劳逃避情绪」在驱动我接更多事**？
5. **写进 life-os 的 `经历流水账.md`**，`[决策]` 格式：
   ```
   YYYY-MM-DD · [决策] 健康事件名
   选择：做 / 不做
   理由：1-2 句（用用户自己的判断框架）
   回看：YYYY-MM-DD（三个月后）
   ```

## 健康场景会触发的常见盲区（事先标好，给联动用）

以下是常见模式，用户实际的盲区以其 `模式与盲点.md` 为准：

- ⚠️ **健康/数字回避**（财务回避的姊妹）：不想看体检报告 / 看到红字想拖
- ⚠️ **用过劳逃避情绪**：明知该休息但还在接事
- ⚠️ **精力分散倾向**：身体已经累了还在加新事
- ✅ **主动在意身体**（正向锚点）：开始记录症状、主动看医生——"提醒未来的自己"

每次健康决策时，AI 应该**主动扫**这几条，不要等用户自己说。

## 关于用户健康（速查，读文件前先看这里）

> 注意：以下是通用框架。用户实际的健康基线在其 `健康档案.md` 里。

**常见高风险场景**：
- 脑力工作者 + 大量看屏幕 → 视力/颈椎
- 高压工作 → 睡眠/熬夜
- 用过劳逃避情绪 → 身体地基被消耗

**建议建立的基本盘**：睡眠 / 运动 / 视力 / 颈椎（可在 `健康档案.md` 里记录用户自己的目标）
