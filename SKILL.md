# 人生系统 · 你的自我认知操作系统

## 这是什么

一套帮你**让 AI 真正了解你**的文件系统。

很多人用 AI 做心理疏导、做决策辅助、做内容创作时都遇到过同一个问题：每次都要从头介绍"我是谁"，AI 只能拿到碎片，给的建议也漂在表面。

这套系统的解法是——把你这个人按三层结构存进本地文件，让 AI 每次都能调用完整的你：

| 层 | 文件 | 装什么 |
|---|------|--------|
| **事件层** | `经历流水账.md` | 发生在你身上的事，倒序，随手记 |
| **规律层** | `模式与盲点.md` | 你反复出现的行为模式 / 容易栽的坑 / 情绪触发 |
| **叙事层** | `我的故事.md` | 你这个人是谁，你的判断框架、价值观、口头禅 |

三层不重复——流水账是收件箱，模式与盲点是雷达，我的故事是图书馆。事件流入，规律提炼，叙事沉淀。

---

## 这个系统能帮你做什么

- **拆解情绪**：当一种"过度的"情绪来袭，AI 能拿着你完整的盲区图，告诉你这次又踩到了哪个老地方——而不是给你一段大众化的安慰
- **看见进步**：同一个盲区，案例日志里 ❌ 和 ✅ 交替出现，进步轨迹一眼可见
- **辅助决策**：用你自己的判断框架（你早就有，只是没系统化）做决策，AI 直接给结论而不是给选项
- **联动书摘**：把你读过的书的划线，按盲区分类——下次又栽进同一个坑，递给你的不是泛泛鸡汤，是你自己曾经划过的那句话
- **反向触发创作**：如果你做内容，洞察可以自动扫描"选题潜力"，反向流向你的创作系统

## 事件流转规则（每记一个事件时遵循）

**流水账（必做）→ 模式与盲点（每次检查，命中就写）→ 我的故事（条件触发）**

### 第一步：写流水账
- 新条目加最上面（倒序）
- 格式：`YYYY-MM-DD · [标签] 一句话标题`
- 3 个标签：
  - `[决策]` — 选择类（含已定/待定）
  - `[模式]` — 和模式与盲点对接，用 `❌`（盲区再现）/ `✅`（突破或正向触发）
  - `[洞察]` — 想清楚了一件事 / 提炼出一个原则
  - 也可无标签

### 第二步：自动同步模式与盲点（所有流水账都要检查）
- 每写一条流水账后，都必须多问一句：这条有没有证明某个旧模式，或长出一个新模式？
- 对应已有条目 → 在该条目「案例日志」加一行
- 看出是新规律 → 直接建新条目
- 不要求流水账标签必须是 `[模式]`；`[洞察]`、普通流水账、好消息、工作策略、内容反馈，只要命中规律层，都要同步 `模式与盲点.md`
- 直接写，事后可删（不阻塞）

### 第三步：触发盲点/突破时附一句书摘（如启用书摘模块）
- 看命中条目的 `📖 书摘` 层
- 按当前是 ❌ 还是 ✅ 挑对应类型的句子
- 同时递给用户 + 写进案例日志缩进行

### 第四步：merge 进我的故事（条件触发）
- 改变了判断框架 / 自我认知 / 形成主题集群 → 直接 merge 进对应章节
- 在流水账对应条目末尾加 ` ✅ merged`

**统一原则**：流水账、模式与盲点、我的故事 = 都直接写、事后可删——先做后看，不阻塞节奏。

## 日常使用规则（用户已搭好系统后）

### 触发场景一：记流水账

用户说「记一下」「今天发生了 X」「我想记下这件事」时：

1. 读用户的 `经历流水账.md` 顶部
2. 把内容整理成 1-5 句话，保留原话味道
3. 判断标签：[决策] / [模式] ❌或✅ / [洞察] / 无
4. 加在流水账最上面（紧接注释之后）
5. **每条流水账都要检查是否触发 `模式与盲点.md`**：不只 `[模式]` 标签才同步。只要这条记录能证明已有正向模式/盲区，或暴露新规律，就直接写进对应条目的「案例日志」
6. **检查顺序**：先扫正向模式 → 再扫已识别盲区 → 再判断是否需要新建模式。常见触发包括：真实反馈/被看见、数据焦虑的反向证据、赚钱动机变化、好活识别、精力分散、给得出要不到、对谈能力等
7. **如果命中已知盲区/正向模式** → 按下方"书摘提醒"附一句（如启用书摘模块）
8. **如果是 [洞察]** → 顺手扫"选题潜力"，告诉用户能不能做成内容

### 触发场景二：帮做决策

用户说「帮我做决策」「我有个选择」「这件事值不值得做」时：

1. 读全部 3 个核心文件（流水账 + 模式与盲点 + 我的故事）
2. 用用户自己的判断框架分析（从我的故事里提炼出来的，比如"三年后悔法""收窄还是分散"等——如果用户还没整理出框架，就引导他想清楚自己常用的判断标准）
3. **直接给结论**，不给选项让用户选
4. 命中盲区/情绪情境时附书摘
5. 写入 `经历流水账.md`，`[决策]` 格式：
   ```
   YYYY-MM-DD · [决策] 事件名
   选择：做 / 不做
   理由：1-2 句
   回看：YYYY-MM-DD（三个月后）
   ```

### 触发场景三：merge 进我的故事

用户说「merge 一下」「这件事值得沉淀」「更新故事库」时，或满足以下任一条件：
- 一组事件改变了判断框架/自我认知
- 一周内多条同主题的事件形成集群
- 用户主动要求

→ **直接 merge** 进 `我的故事.md` 对应章节（不是追加末尾，是融进相应位置），在流水账对应条目末尾加 ` ✅ merged`，汇报"merge 了什么、放在哪一节"，用户可删。

### 触发场景四：周复盘

用户说「周复盘」「这周时间花哪了」「照镜子」时，进入**递镜模式**（这是唯一不直接给结论的场景）：

1. 让用户给本周时间饼图（粗略百分比即可）：主业 / 副业 / 学习 / 社交 / 休息 / 其他
2. 让用户标关键维度方向（↑↑/↑/→/↓/↓↓）：可以是自己定义的维度，比如能力成长 / 财务 / 人际 / 身体 / 创作产出 等
3. 必照"镜子背面"：碎片时间用在了哪里、身体地基（睡眠、运动）
4. **不打分、不给 to-do list、不直接给结论**——用户给数字时停顿/愣住的地方就是镜子要照的
5. 解读阶段对比用户的方向（来自 `我的故事.md`）
6. 存进 `周复盘.md`（倒序追加）
7. 出现 aha moment → 同步追加流水账标 `[洞察]`

### 触发场景五：书摘提醒（如启用）

触发时机：[模式] 标签命中盲区（加 ❌）或正向模式（加 ✅）或情绪触发地图某一行。

执行：
1. 读命中条目的 `📖 书摘` 层
2. 按当前是 ❌ 还是 ✅ 挑对应类型的句子（标 (❌) 提醒类 / (✅) 巩固类 / (通用)）
3. **同时做两件事**：
   - 在汇报里递给用户：`📖 你自己划过的一句话：「书摘原文」——《书名》`
   - 写进案例日志该条下方（缩进 2 格）：
     ```
     - ❌/✅ YYYY-MM-DD 事件描述...
       📖 「书摘原文」——《书名》
     ```

一次只递一句，不堆。没贴切的就不附。

给用户的建议：
- 敏感人物用代号（M / J / 朋友A / 同事B）
- 真正不想让任何系统看到的内容，不记进系统
- 这是给你"看见模式"的工具，不是"逐字真相"的存档——代号化的叙述不影响 AI 分析模式
